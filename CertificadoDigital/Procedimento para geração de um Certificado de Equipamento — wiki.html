Procedimento para geração de um Certificado de Equipamento

Para acessar o Gertoken, é necessário que seja passado para ele um certificado digital que identificará o servidor que faz a solicitação do token. Para isso, é necessário que esse certificado esteja registrado no servidor Gertoken que se deseja acessar. Segue abaixo um passo a passo de como se gerar o certificado digital utilizando o OpenSSL e registrar o acesso ao servidor Gertoken.

1- Criar uma chave privada para o servidor. Utilize o comando abaixo para gerar a chave privada do servidor. A chave criada leva em consideração o equipamento onde a mesma foi gerada, sendo necessário uma chave específica por servidor que for acessar o Gertoken. Nesse exemplo chamaremos a chave privada de chaveVIA.key

     openssl genrsa -out chaveVIA.key 1024 


O arquivo texto chaveVIA.key gerado terá o seguinte formato:

2- Criar a solicitação de certificado. O comando abaixo gera uma requisição de certificado que deverá ser informado ao se solicitar o certificado do orgão expedidor. Chamaremos a requisição de requisiçãoVIA.csr

     openssl req -new -key chaveVIA.key -out requisiçãoVIA.csr 


A seguir serão pedidas as seguintes informações já preenchidas nesse exemplo.

    into your certificate request.

    What you are about to enter is what is called a Distinguished Name or a DN.

    There are quite a few fields but you can leave some blank

    For some fields there will be a default value,

    If you enter '.', the field will be left blank.

    -----

    Country Name (2 letter code) [GB]:BR

    State or Province Name (full name) [Berkshire]:Rio de Janeiro

    Locality Name (eg, city) [Newbury]:Rio de Janeiro

    Organizational Unit Name (eg, section) []:Receita Federal do Brasil-RFB

    Email Address []: nome.usuario@serpro.gov.br

    to be sent with your certificate request

    A challenge password []:

Para verificar o conteúdo da solicitação também pode ser utilizado o comando abaixo

    openssl req -noout -text -in suite.desenv.csr


3- Fazer a solicitação do certificado no órgão expedidor. Envie o conteúdo do arquivo CSR na solicitação do certificado A1 de Equipamento em https://ccdhom.serpro.gov.br/acserprorfb/
O procedimento detalhado é encontrado em http://10.200.227.45:7070/portaldebhe/sistemas-e-componentes/de6we/e-cac/area-publica/manual-desenvolvedor/testes-das-aplicacoes?searchterm=certificado

4- Fazer o download do certifcado. Assim que a requisição for atendida, retorne ao endereço onde foi feita a requisição para fazer o download do certificado. É necessário copiar o conteúdo do campo Certificado X.509v3 (certificado em formato PEM) para um arquivo .cer (Ex.: certificadoVIA.cer)

5- Converter a chave privada do formato PEM para o formato DER. Para tal use o comando abaixo para gerar o o arquivo .der (Ex.: certificadoVIA.der)

    openssl pkcs8 -topk8 -nocrypt -in chaveVIA.key -inform PEM -out certificadoVIA.der -outform DER



6- Criar uma keystore. A aplicação do Gertoken utiliza uma keystore com o certificado para identificar as requisições, portanto é necessário gerar essa keystore. Essa keystore pode ser gerada utilizando através da ferramenta ImportKey.java, encontrada em http://www.agentbob.info/agentbob/79-AB.html ou utilizando o comando

    wget http://www.agentbob.info/agentbob/80/version/default/part/AttachmentData/data/ImportKey.java


6.1- Compile a ferramenta ImportKey.java

    javac ImportKey.java


6.2- Adicionar a chave privada ao keystore. A keystore é um arquivo binário que devera ser carregado pela aplicação para que ela possa ser enviada para o servidor Gertoken. O comando abaixo irá gerar o arquivo keystoreVIA.keystore com o alias keystoreVIA. O alias é o nome que deverá ser utilizado para referenciar a keystore na aplicação

    java -Dkeystore=keystoreVIA.keystore ImportKey certificadoVIA.der certificadoVIA.cer keystoreVIA

Para se verificar o conteúdo da keystore gerada, utilize o comando abaixo. Após a execução desse comando, será pedido a senha da keystore. A ferramenta ImportKey.java utiliza a senha changeit para todas as keystores geradas por ele. Essa senha pode ser mudada alterando o código do ImportKey.java
 
    keytool -list -v -keystore keystoreVIA.keystore



7- Registrar o certificado no servidor Gertoken. Ao terminar esse processo, o certificado (certificadoVIA.cer) deve ser registrado no servidor Gertoken e a chave privada deve ser incluída no servidor que fará acessso ao GERTOKEN, para que eles possam validar o acesso. Para isso deve-se enviar essas informações para os responsáveis pelo servidor do Gertoken e o servidor onde consta o serviço.
